# API Security Risk Analysis

## Overview
This project analyzes API security risks using a public test API (JSONPlaceholder).

## Tools Used
- Postman
- Browser DevTools
- MS Word / PDF

## API Tested
- JSONPlaceholder (https://jsonplaceholder.typicode.com)

## Methodology
- Reviewed API documentation
- Tested API endpoints (GET requests)
- Checked request/response headers
- Identified risks

## Risks Identified
- No authentication (High)
- Excessive data exposure (Medium)
- No rate limiting (Medium)
- Missing input validation (Low)

## Recommendations
- Implement authentication (API keys / JWT)
- Limit data exposure
- Add rate limiting
- Validate all inputs
- Use HTTPS for secure communication
